## Overview

The Properties API provides comprehensive endpoints for managing real estate listings. All property operations require authentication except for listing and retrieving individual properties.

**Base Path:** `/api/properties`

---

## List Properties

<CodeGroup>
```typescript GET /properties
const response = await api.properties.list({
  query: "departamento",
  propertyType: ["casa", "departamento"],
  listingType: ["venta"],
  minPrice: 50000,
  maxPrice: 200000,
  bedrooms: 2,
  bathrooms: 1,
  location: "Buenos Aires",
  page: 1,
  limit: 10
});
```
</CodeGroup>

Retrieve a paginated list of properties with optional filtering.

### Query Parameters

<ParamField query="query" type="string">
  Search text to match in property title or description
</ParamField>

<ParamField query="propertyType" type="string[]">
  Filter by property types (casa, departamento, ph, terreno, local, galpon, oficina, campo, deposito, bodega, consultorio, fondo-comercio, hotel, cochera)
</ParamField>

<ParamField query="listingType" type="string[]">
  Filter by operation types (venta, alquiler, alquiler-temporal, permuta)
</ParamField>

<ParamField query="status" type="string[]">
  Filter by property status (activo, pendiente, vendido, alquilado, pausado)
</ParamField>

<ParamField query="minPrice" type="number">
  Minimum price filter
</ParamField>

<ParamField query="maxPrice" type="number">
  Maximum price filter
</ParamField>

<ParamField query="bedrooms" type="number">
  Minimum number of bedrooms
</ParamField>

<ParamField query="bathrooms" type="number">

</ParamField>

<ParamField query="location" type="string">
  Location search (city, province, neighborhood)
</ParamField>

<ParamField query="page" type="number" default="1">
  Page number for pagination
</ParamField>

<ParamField query="limit" type="number" default="10">
  Number of items per page

## Create Property

<CodeGroup>
```typescript POST /properties
const formData = new FormData();
formData.append("title", "Casa en Palermo");
formData.append("description", "Hermosa casa con jardín");
formData.append("price", "250000");
formData.append("propertyTypeId", "1");
formData.append("operationTypeId", "1");
formData.append("bedrooms", "3");
formData.append("bathrooms", "2");
formData.append("images", imageFile);

const response = await api.properties.create(formData);
```
</CodeGroup>

Create a new property listing. Requires admin authentication.

### Request Body

Multipart form data with the following fields:

<ParamField body="title" type="string" required>
  Property title (max 255 characters)
</ParamField>

<ParamField body="description" type="string" required>
  Detailed property description
</ParamField>

<ParamField body="price" type="number" required>
  Property price
</ParamField>

<ParamField body="propertyTypeId" type="number" required>
  Property type ID (from metadata)
</ParamField>

<ParamField body="operationTypeId" type="number" required>
  Operation type ID (from metadata)
</ParamField>

<ParamField body="bedrooms" type="number" required>
  Number of bedrooms
</ParamField>

<ParamField body="bathrooms" type="number" required>
  Number of bathrooms
</ParamField>

<ParamField body="surfaceCoveredM2" type="number">
  Covered surface area in m²
</ParamField>

<ParamField body="images" type="File[]">
  Property images (multiple files supported)
</ParamField>

<ParamField body="city" type="string" required>
  City name
</ParamField>

<ParamField body="province" type="string" required>
  Province/state name
</ParamField>

<ParamField body="street" type="string">
  Street name
</ParamField>

<ParamField body="latitude" type="number">
  Latitude coordinate
</ParamField>

<ParamField body="longitude" type="number">
  Longitude coordinate
</ParamField>

<ParamField body="features" type="string">
  Comma-separated feature IDs
</ParamField>

<ParamField body="tags" type="string">
  Comma-separated tag IDs
</ParamField>

### Response

<ResponseField name="success" type="boolean" required>

</ResponseField>

<ResponseField name="data" type="object" required>

## Error Responses

All endpoints may return the following error responses:

<ResponseField name="error" type="object">
  <Expandable title="Error structure">
    <ResponseField name="message" type="string">
      Human-readable error message
    </ResponseField>
    
    <ResponseField name="code" type="string">
      Error code (e.g., "UNAUTHORIZED", "NOT_FOUND", "VALIDATION_ERROR")
    </ResponseField>
  </Expandable>
</ResponseField>

### Common Status Codes

- `200` - Success
- `400` - Bad Request (invalid parameters)
- `401` - Unauthorized (authentication required)
- `403` - Forbidden (insufficient permissions)
- `404` - Not Found (property doesn't exist)
- `500` - Internal Server Error
